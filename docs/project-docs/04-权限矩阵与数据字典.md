# 权限矩阵与数据字典

## 1. 文档目的

本文档用于明确 HR Agent 项目的角色权限、核心数据对象、字段含义、枚举值和数据治理规则。研发、测试和业务验收应以本文档作为权限与数据口径依据。

## 2. 角色定义

| 角色 | 说明 |
| --- | --- |
| 超级管理员 | 管理用户、角色、邮箱配置和系统级资源 |
| HR 负责人 | 查看招聘、候选人、面试和统计数据 |
| HR 专员 | 执行招聘流程，包括 JD、简历、面试 |
| 培训管理员 | 管理培训资料、试卷和考试结果 |
| 普通员工 | 使用知识问答或参与考试 |
| 候选人/考试参与者 | 通过分享链接参与考试，不进入后台系统 |

### 3.2 可选扩展权限

如试点企业希望扩大使用范围，可在需求变更评审后开启以下扩展权限：

| 扩展项 | 可开放角色 | 风险 |
| --- | --- | --- |
| 培训管理员触发招聘 Agent 工具 | 培训管理员 | 可能误操作招聘数据 |
| HR 负责人管理知识库 | HR 负责人 | 可能修改企业知识资料 |
| HR 专员管理知识库 | HR 专员 | 可能上传不合规资料 |
| HR 负责人查看考试结果 | HR 负责人 | 可能扩大培训结果可见范围 |
| 普通员工查看统计看板 | 普通员工 | 可能暴露业务运营数据 |

扩展权限必须记录开启原因、影响范围、审批人和回退方式。

## 5. 数据治理要求

| 数据类型 | 敏感等级 | 管理要求 |
| --- | --- | --- |
| 用户账号和密码 | 高 | 密码哈希存储，账号权限控制 |
| 简历和候选人信息 | 高 | 限制访问、明确保留期限、支持删除 |
| 面试方案和评分结果 | 高 | 仅授权 HR/管理者访问 |
| 企业文档和知识库 | 中到高 | 按文档敏感度控制访问 |
| 试卷和考试结果 | 中 | 控制查看和导出权限 |
| 邮箱配置 | 高 | 仅超级管理员访问，不暴露密码 |
| AI 对话记录 | 中 | 避免写入不必要敏感信息 |

## 6. 数据保留建议

| 数据 | 建议保留策略 |
| --- | --- |
| 简历文件 | 试点期按业务要求保留，结束后清理或归档 |
| 简历评分记录 | 与招聘岗位周期一致 |
| JD 和评分标准 | 可长期保留用于复用 |
| 面试方案 | 与候选人流程周期一致 |
| 知识库文档 | 按企业知识管理规则保留 |
| 考试结果 | 按培训合规要求保留 |
| 日志 | 至少保留一个排障周期，生产按企业规范 |
