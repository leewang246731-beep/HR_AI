# 需求文档

## 1. 文档目的

本文档说明 HR Agent 项目要做什么、不做什么、用户需要什么、业务流程如何闭环、功能边界在哪里。文档面向产品、研发、测试、业务负责人和上线验收人员。

## 2. 需求范围

### 2.1 范围内

- 用户注册、登录、个人信息、JWT 鉴权。
- 用户管理、角色管理、超级管理员权限控制。
- HR Agent 对话入口、意图识别、受控工具编排、SSE 流式响应。
- JD 生成、保存、查询、修改、删除。
- 简历评分标准生成、保存、查询、修改、删除。
- 简历上传、解析、AI 评分、历史记录、状态更新、删除、导出。
- 面试方案生成、保存、查询、修改、删除。
- 文档上传、处理、查询、删除、文档切片查看。
- 知识库管理、知识助手问答、知识库自动选择。
- 试卷生成、试卷管理、考试分享、考试提交、考试结果查看和导出。
- 邮箱配置、连接测试、邮件抓取、日志查看。
- 工作台统计、招聘趋势、培训完成情况、近期活动。

### 2.2 范围外

- 招聘渠道发布与渠道效果分析。
- 候选人 CRM、Offer 审批、背调、入职办理。
- 企业组织架构、绩效、薪酬、考勤等完整 HRIS 功能。
- 自动录用、自动淘汰、自动发送不可撤销通知。
- 面向候选人的完整招聘门户。
- 企业级监控、CI/CD、服务发现、灰度平台建设。
- 大规模多租户 SaaS 计费、套餐、租户隔离和商业化运营后台。

## 3. 用户角色与权限需求

| 角色 | 核心诉求 | 主要权限 |
| --- | --- | --- |
| 超级管理员 | 管理系统、用户、角色、邮箱配置 | 用户管理、角色管理、邮箱配置、全部业务数据维护 |
| HR 负责人 | 查看招聘培训进展，保证流程质量 | 查看统计、招聘结果、培训结果、关键业务数据 |
| HR 专员 | 执行招聘任务 | JD、评分标准、简历筛选、面试方案、邮件草稿 |
| 培训管理员 | 生成和管理考试 | 文档上传、试卷生成、考试管理、结果查看 |
| 普通员工 | 查询知识或参与考试 | 知识问答、考试分享页面 |
| 候选人/考试参与者 | 完成考试或测评 | 通过分享链接访问考试页面 |

权限要求：

- 登录后才能访问主系统页面。
- 用户管理、角色管理、邮箱配置等敏感操作仅允许超级管理员访问。
- 普通业务功能至少要求已认证用户。
- 考试分享页面可按业务需要允许免登录访问。
- 后端接口必须以 JWT 和角色校验为准，前端菜单隐藏不能作为唯一权限控制。

## 4. 核心用户需求

### 4.1 HR 专员

- 我希望输入岗位需求后自动生成结构完整的 JD，以减少重复撰写。
- 我希望系统同步生成评分标准，使后续简历筛选口径一致。
- 我希望批量上传简历并自动评分，快速定位高匹配候选人。
- 我希望看到评分理由和维度，而不是只有一个总分。
- 我希望基于候选人和岗位自动生成面试方案，便于面试官使用。
- 我希望通过 Agent 直接描述任务，由系统判断去哪个功能或执行哪个流程。

### 4.2 HR 负责人

- 我希望看到招聘相关统计，了解当前 JD、简历筛选、面试等进展。
- 我希望流程产物可追踪、可复用、可删除。
- 我希望 AI 结果保留人工确认空间，避免系统替 HR 做最终判断。

### 4.3 培训管理员

- 我希望上传培训文档后，系统可以自动生成试卷。
- 我希望配置题型、难度、题量、分值等信息。
- 我希望发布考试链接并查看考试结果。

### 4.4 系统管理员

- 我希望能管理用户、角色和邮箱配置。
- 我希望系统有健康检查、错误提示和日志，便于定位问题。
- 我希望敏感配置不被普通用户访问。

## 5. 业务流程需求

### 5.1 招聘主流程

1. HR 输入岗位需求。
2. 系统解析岗位名称、地点、薪资、经验、学历、技能要求等信息。
3. 系统调用 JD 生成能力输出 JD 初稿。
4. 系统生成与 JD 匹配的评分标准。
5. HR 保存或编辑 JD 与评分标准。
6. HR 上传候选人简历。
7. 系统解析简历，结合 JD 和评分标准输出评分、匹配度和理由。
8. HR 查看候选人排序和评分详情。
9. HR 选择候选人生成面试方案。
10. HR 根据面试方案安排面试或准备邮件通知。

### 5.2 Agent 对话流程

1. 用户在 HR Agent 输入自然语言。
2. 系统归一化附件信息并识别意图。
3. 系统判断动作：普通回复、调用工具、追问缺失信息。
4. 若可执行，系统根据工具注册表调度 JD、简历筛选、面试方案、试卷生成、资源删除或邮件 Skill。
5. 若缺少前置条件，系统返回缺失字段、建议和对应页面入口。
6. 若任务耗时较长，系统通过 SSE 返回执行进度。
7. 前端展示步骤、结果、产物和下一步建议。

### 5.3 知识库问答流程

1. 用户上传文档。
2. 系统保存文件并处理文本内容。
3. 系统将文档切片并向量化，写入 PostgreSQL + pgvector。
4. 用户在知识助手输入问题。
5. 系统进行知识库选择、查询增强、向量召回、文本匹配和 rerank。
6. 系统将相关上下文交给 LLM 生成回答。
7. 用户查看回答和来源上下文。

### 5.4 培训考试流程

1. 培训管理员上传参考文档。
2. 管理员输入试卷配置或通过 Agent 表达考试需求。
3. 系统解析考试意图，生成试卷。
4. 管理员保存试卷并生成分享链接。
5. 考试参与者通过链接答题并提交。
6. 系统记录考试结果，支持查看和导出。

### 5.5 系统管理流程

1. 超级管理员登录系统。
2. 管理用户信息、角色和权限。
3. 配置邮箱服务并进行连接测试。
4. 查看邮件抓取日志和最近邮件。
5. 通过健康检查和日志确认系统状态。

## 6. 功能需求

### 6.1 认证与用户管理

- 支持用户注册、登录、刷新 Token、获取当前用户。
- 支持用户列表、详情、更新、删除。
- 支持管理员创建用户、分配角色、创建角色、删除角色。
- 支持超级管理员角色校验。
- 登录失败、Token 过期、无权限时返回明确错误。

详细规则：

- 注册时用户名、邮箱应唯一。
- 登录成功后返回访问令牌和令牌类型。
- Token 失效后前端应引导用户重新登录。
- 普通用户不得调用用户列表、角色管理、邮箱管理等管理员接口。
- 超级管理员角色以系统内角色或 `is_superuser` 校验为准，不能只依赖前端菜单控制。

### 6.2 HR Agent

- 支持普通对话接口和流式对话接口。
- 支持附件元数据传入。
- 支持对 JD、简历筛选、面试方案、试卷生成、资源删除、邮件通知等意图的识别。
- 支持根据前置条件返回缺失信息。
- 支持输出步骤、产物、建议、路由和执行状态。
- 邮件类动作只能生成草稿或进入确认流程，不允许未经确认自动发送。

详细规则：

- 普通问候、解释系统能力、无明确任务的问题应走普通回复，不应误触发工具。
- 用户目标明确但缺少前置条件时，应返回缺失字段，而不是直接失败。
- 对删除类、发送类、影响外部对象的动作，应要求确认。
- Agent 返回结构应包括 `message`、`intent`、`route`、`steps`、`artifacts`、`suggestions`、`requires_confirmation`、`missing_fields`。
- 流式接口应持续推送步骤或内容片段，前端需要展示执行进度。

### 6.3 JD 与评分标准

- 支持根据岗位需求生成 JD。
- 支持保存、查询、更新、删除 JD。
- 支持生成并保存评分标准。
- 支持按岗位、状态或关键字查询历史记录。
- 支持 JD 与评分标准作为后续简历评分输入。

详细规则：

- JD 必填字段至少包括标题和正文内容。
- 可选字段包括部门、地点、薪资范围、经验、学历、岗位类型、技能列表、状态、元数据。
- JD 默认状态为草稿，业务可根据需要扩展为已发布、归档等状态。
- 评分标准默认总分为 100 分。
- 评分标准应支持维度化存储，包括维度名称、权重、满分和评分说明。
- 删除 JD 前应确认是否存在关联评分标准或简历评估记录；如果当前实现未做强约束，验收时需确认业务可接受。

### 6.4 简历筛选

- 支持 PDF、DOC、DOCX 等简历格式。
- 支持上传文件大小限制。
- 支持自动解析简历内容。
- 支持结合 JD 或评分标准进行匹配评分。
- 支持查看评分结果、评分指标、AI 建议和历史记录。
- 支持状态更新、删除和压缩包导出。

详细规则：

- 简历评估必须关联一个 JD。
- 评分标准可选；若未选择，系统可使用 JD 内容或默认评估逻辑。
- 评估结果至少包括候选人姓名、应聘岗位、工作年限、教育背景、总分、评分维度和评分理由。
- 支持候选人状态更新，状态枚举应在产品验收时固定。
- 同一候选人重复上传的处理规则需在上线前确认：允许重复、覆盖旧记录或提示用户选择。
- AI 评分结果不作为淘汰或录用的最终依据。

### 6.5 面试方案

- 支持基于候选人简历、JD 和评分结果生成面试方案。
- 支持保存、查询、更新、删除面试方案。
- 面试方案应包含面试重点、问题建议、考察维度、评分建议。

详细规则：

- 面试方案必须关联一条简历评估记录。
- 创建时必须包含候选人姓名、候选人岗位和方案正文。
- 方案内容应覆盖技术能力、项目经验、岗位匹配度、风险点和追问建议。
- 面试方案保存后允许 HR 编辑，不要求完全采用 AI 原文。

### 6.6 知识库与知识助手

- 支持文档上传、处理、切片和删除。
- 支持知识库集合创建、更新、删除。
- 支持知识库问答、自动选择知识库。
- 支持查询增强、向量召回、文本匹配、rerank。
- 对解析失败、空文档、无相关结果应返回明确提示。

详细规则：

- 知识库名称必填，支持描述、公开状态、可搜索状态、分类、标签和元数据。
- 文档上传可绑定知识库，也可以作为未归类文档保存。
- 文档处理后应保存提取文本、文件哈希、文件大小、MIME 类型和切片信息。
- 知识问答应优先基于知识库内容回答；无相关内容时应提示不确定。
- 用户删除文档时，应同步考虑文档切片和向量数据清理。

### 6.7 试卷与考试

- 支持基于文档生成试卷。
- 支持配置题型、难度、题量、分值。
- 支持试卷列表、详情、更新、删除。
- 支持考试分享链接。
- 支持提交答案、记录考试结果、查看和导出结果。

详细规则：

- 试卷生成请求应包括标题、科目、总分，可选难度、时长、题型、题量、知识文件和特殊要求。
- 试卷保存后应支持列表、详情、更新、删除。
- 考试提交应记录考试 ID、学生姓名、部门、答案和试卷内容。
- 分享链接是否免登录访问，应由业务验收确认。
- 若试卷已被提交答题，是否允许修改或删除，需在上线前明确。

### 6.8 邮箱配置

- 支持邮箱配置新增、查询、更新、删除。
- 支持连接测试。
- 支持手动抓取邮件、查看抓取日志和最近邮件。
- 邮箱配置仅允许超级管理员操作。

### 6.9 统计看板

- 支持工作台概览。
- 支持招聘趋势、培训完成情况和近期活动查询。

## 6.10 页面级需求

| 页面 | 主要操作 | 关键状态 | 异常提示 |
| --- | --- | --- | --- |
| 登录页 | 登录、跳转注册 | 未登录、登录中、登录成功 | 账号密码错误、Token 异常 |
| 工作台 | 查看统计和活动 | 加载中、正常、无数据 | 统计接口失败 |
| HR Agent | 输入消息、上传附件、执行任务 | 等待输入、执行中、需确认、完成、失败 | 意图不明确、缺少字段、工具失败 |
| JD 生成 | 输入需求、生成、保存、编辑、删除 | 草稿、已保存、生成中 | 缺少岗位信息、生成失败 |
| 简历筛选 | 上传简历、选择 JD、查看评分、导出 | 上传中、评估中、完成、失败 | 格式错误、大小超限、解析失败 |
| 智能面试 | 选择候选人、生成方案、保存 | 待选择、生成中、已保存 | 缺少候选人或评分记录 |
| 试卷生成 | 上传资料、配置试卷、生成、保存 | 配置中、生成中、已保存 | 文档不可用、题型配置错误 |
| 考试管理 | 查看试卷、分享、结果 | 未发布、可分享、有结果 | 试卷不存在、结果导出失败 |
| 知识库管理 | 创建知识库、上传文档、处理、删除 | 未处理、处理中、已处理 | 文档解析失败 |
| 知识问答 | 输入问题、选择知识库、查看回答 | 查询中、已回答、无结果 | 无相关知识、LLM 失败 |
| 用户管理 | 创建、编辑、删除、分配角色 | 正常、禁用、删除确认 | 权限不足、用户重复 |
| 邮箱管理 | 配置、测试、抓取、查看日志 | 未配置、测试中、可用、失败 | SMTP/IMAP 连接失败 |

### 6.10.1 页面字段与交互规则

#### 登录页

| 项 | 规则 |
| --- | --- |
| 输入字段 | 用户名或邮箱、密码 |
| 主按钮 | 登录 |
| 校验规则 | 账号和密码不能为空 |
| 成功行为 | 保存 Token 和用户信息，跳转工作台 |
| 失败行为 | 展示账号密码错误或认证失败提示，不清空账号 |

#### JD 生成页

| 项 | 规则 |
| --- | --- |
| 输入字段 | 岗位名称、部门、地点、薪资范围、经验要求、学历要求、岗位类型、技能、岗位需求描述 |
| 必填字段 | 岗位名称或岗位需求描述至少填写一项；保存时 JD 标题和正文必填 |
| 主按钮 | 生成 JD、保存 JD、编辑、删除 |
| 生成中状态 | 禁用重复生成按钮，展示流式输出或加载状态 |
| 保存成功 | 进入已保存状态，列表可查询，保留可编辑能力 |
| 删除规则 | 删除前二次确认；删除成功后列表移除 |
| 异常 | 缺少输入提示补充；生成失败保留原输入并允许重试 |

#### 评分标准页

| 项 | 规则 |
| --- | --- |
| 输入字段 | JD 内容、岗位标题、总分、评分维度、权重、评分说明 |
| 必填字段 | 标题、评分标准正文 |
| 绑定规则 | 可绑定 JD；绑定后用于后续简历评估 |
| 编辑规则 | 支持调整维度、权重和说明 |
| 校验规则 | 总分默认 100；维度权重口径应在保存前人工确认 |
| 删除规则 | 删除前二次确认 |

#### 简历筛选页

| 项 | 规则 |
| --- | --- |
| 输入字段 | 简历文件、JD、可选评分标准 |
| 必填字段 | 简历文件、JD |

## 6.12 待定业务口径收敛

以下事项在试点默认按本表执行；如企业另有规则，需在需求评审会上变更确认。

| 事项 | 试点默认口径 | 变更影响 |
| --- | --- | --- |
| 考试分享页是否免登录 | 默认允许免登录答题 | 影响前端路由和后端鉴权 |
| 已有答题结果的试卷是否允许修改 | 默认不直接修改，复制为新版本 | 影响试卷版本管理 |
| 同一简历重复上传 | 默认允许重复记录，不覆盖旧记录 | 影响去重和历史记录 |
| JD 删除时存在关联评估 | 默认允许软删除或逻辑下线，不物理删除关联数据 | 影响数据一致性 |
| 候选人状态枚举 | 待复核、进入面试、暂缓、淘汰 | 影响列表筛选和统计 |
| 知识库权限 | 试点默认登录用户可问答，管理权限按角色控制 | 影响普通员工范围 |
| 邮件发送 | 默认只生成草稿或确认后发送 | 影响自动化边界 |

## 7. 非功能需求

### 7.1 性能

- 常规列表查询应在可接受时间内返回，避免前端长时间空白。
- LLM 生成类任务应优先使用流式响应或进度提示。
- 文档上传和简历上传应限制文件大小，当前配置默认最大 10MB。

建议指标：

- 普通列表查询：试点环境下 3 秒内返回。
- 登录和权限校验：试点环境下 2 秒内返回。
- 长文本生成：5 秒内出现首个进度或流式片段。
- 文件上传：应显示上传进度或处理状态。

### 7.2 安全

- 密码必须哈希存储。
- API 使用 JWT Bearer Token 鉴权。
- 生产环境必须替换默认 `SECRET_KEY`、数据库密码和外部 API Key。
- 上传文件应限制格式和大小。
- 邮箱、Dify、LLM 等敏感配置不得暴露给普通用户。

### 7.3 可用性

- 前端菜单应按业务模块组织。
- 表单错误、权限错误、上传失败、生成失败应有明确提示。
- 长任务要有进度反馈。

### 7.4 可维护性

- 后端按 endpoint、schema、model、service 分层。
- 前端 API 请求统一维护在 `src/api`。
- 新增业务功能应补充接口、服务、页面和测试用例。

### 7.5 合规与人工确认

- AI 输出只能作为辅助建议，不作为最终录用、淘汰或评价决定。
- 关键通知和候选人沟通内容应由 HR 审核确认。
- 简历和候选人信息属于敏感个人信息，上线前需确认存储、访问和删除策略。

### 7.6 可观测性

- 后端应记录启动、数据库初始化、外部服务调用失败、权限失败、文件解析失败等关键日志。
- 上线环境应具备日志查询方式。
- 企业生产化阶段建议补充集中日志、告警和调用链追踪。

### 7.7 可恢复性

- 数据库需要定期备份。
- 上传目录需要持久化和备份。
- 上线前应保存旧版本前端资源和后端包或镜像。
- 发生严重故障时应可回滚到上一版本。
